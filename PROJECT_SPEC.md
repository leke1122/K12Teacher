# 项目说明书：AI 高中全科自学辅助系统 (EduMind)

## 1. 项目概述
开发一个 **Web 端单页应用**，面向高中生。核心价值在于通过 AI 大模型深度解析教材 PDF，提供"知识点精讲"与"课本还原"两种沉浸式学习模式，结合智能练习、数据统计与单词记忆，形成"学-练-测-评"闭环。**关键约束：不写死 API Key，数据存储免费，代码结构清晰，UI 对电脑操作不熟练的用户友好（支持手写板输入）。**

## 2. 技术栈与架构 (防屎山准则)
- **前端框架**：Next.js 14 (App Router) + TypeScript (强类型约束，杜绝 any 滥用)。
- **UI 组件库**：Shadcn/ui + Tailwind CSS (原子化 CSS，避免样式冲突)。
- **状态管理**：Zustand (轻量级，区分 `authStore`, `learningStore`, `statStore`)。
- **数据库**：**Supabase** (PostgreSQL)。推荐理由：提供免费层级、内置用户认证（邮箱注册）、自动生成 TypeScript 类型，比纯 Neon 更易管理 Row Level Security (RLS)。
- **AI 大模型（国产）**：
  - **文本模型**：DeepSeek-V3 或 Qwen-Turbo (性价比极高，长上下文支持 PDF 解析)。
  - **视觉/手写模型**：Qwen-VL-Max 或 GLM-4V (用于理解手写板生成的图片)。
  - **集成方式**：前端通过 `@ai-sdk/openai` 兼容库统一调用，**BaseURL 与 API Key 均在 `/settings` 页面由用户动态配置并存入 localStorage**。

---

## 3. 数据库设计 (核心表结构)
*（Cursor 生成迁移脚本时需严格参照）*

- **`subjects`**: 学科表 (id, name, color, icon)。
- **`textbooks`**: 教材表 (id, subject_id, file_name, file_url)。
- **`chapters`**: 章节表 (id, textbook_id, title, start_page, end_page, chapter_index)。
- **`learning_sessions`**: 学习记录表 (id, user_id, chapter_id, mode: 'KNOWLEDGE'|'TEXTBOOK', progress_json, updated_at)。
- **`qa_history`**: 问答对话历史 (id, session_id, role, content, image_url, created_at)。
- **`wrong_questions`**: 错题本 (id, user_id, subject_id, question, correct_answer, user_answer, analysis, difficulty)。
- **`word_list`**: 英语单词表 (id, word, phonetic, meaning, example, memory_tip)。
- **`word_mastery`**: 用户单词掌握 (id, user_id, word_id, mastery_level: 0-5, review_count)。

## 4. 功能模块详细规格 (AI 实现依据)

### 4.1 教材管理与章节解析 (上传 PDF)
- **交互**：用户选择学科 -> 上传 PDF。
- **AI 逻辑**：读取 PDF 文本（使用 `pdf-parse` 库提取）。Prompt 约束："请根据目录页或内容结构，提取所有章节，返回严格 JSON 数组格式: `[{index, title, startPage, endPage}]`"。解析成功后存入 `chapters` 表。
- **视图**：左侧章节树，点击即进入学习模式选择。

### 4.2 理科学习双模式 + 练习 (数学/物理/化学)

#### 模式 A：知识点精讲 (知识图谱)
- **流程**：AI 提取选中章节所有知识点（Prompt: "提取该范围内所有独立考点"）。前端展示知识点列表。
- **交互**：用户点击"开始逐个击破" -> AI 输出知识点讲解（100-200字） -> AI 自动生成 1 道选择题/判断题 -> 用户作答。
- **纠错闭环**：答对 -> 标记掌握，进入下一知识点。答错 -> **AI 递归细化讲解**（Prompt: "用户答错了，请换一种更通俗的比喻解释该知识点"）-> 重新出题，直至正确。
- **注意**：每次互动均需携带该章节的上下文，但只取前 3 轮对话历史防止 Context 爆炸。

#### 模式 B：还原课本 (逐页逐句拆解)
- **流程**：根据 `startPage` 和 `endPage`，提取文本。
- **交互**：AI 按"页"为单位输出原文核心句 -> 针对该句生成"引导式填空或简答" -> 用户回答正确才显示下一句。
- **进度记录**：必须记录当前所处的 **页码** 和 **句段索引**。

#### 模式 C：章节练习 (综合智能出题)
- **约束**：AI 生成 **简单/中等/困难** 各 10 道题（共 30 道）。
- **题型**：选择题 (60%)、填空题 (30%)、计算题 (10%，用户只需列公式或输入答案)。
- **范围**：Prompt 必须包含"请包含当前章节及之前所有章节的知识点"。

### 4.3 手写板与键盘双输入 (易用性核心)
- **硬件交互**：系统检测到用户连接手写板（数位板），在前端提供 **Canvas 画板** 组件。
- **流程**：用户手写答案 -> 点击"识别" -> Canvas 转 Base64 图片。
- **AI 识别**：调用 **Qwen-VL** 视觉模型，Prompt: "请识别图片中的手写数学公式/英文/汉字，并转换为 LaTeX 或纯文本输出，不要额外解释。"
- **降级方案**：若用户打字熟练，可直接使用键盘输入框，两者在对话流中平等。

### 4.4 学习记录与断点续学 (保存进度)
- **保存机制**：在任何学习界面（知识点或课本模式），顶部悬浮"保存进度"按钮。
- **记录内容**：`progress_json` 存储结构化数据，例如 `{"mode":"KNOWLEDGE", "currentPoint": "牛顿第二定律", "step": 3}` 或 `{"mode":"TEXTBOOK", "page": 45, "sentenceIndex": 2}`。
- **恢复机制**：首页"学习记录"列表，点击某条记录 -> 大模型接收该 JSON 和历史对话上下文 -> **AI Prompt 总结前情**："你现在将接着上次的内容继续，上次讲到了...，现在开始下一步。" 无缝衔接。

### 4.5 数据统计与薄弱点分析 (学情雷达)
- **时长统计**：监听路由切换，计算每个学科 `focus_time`，每日汇总存入 Supabase。
- **薄弱点分析**：**核心创新**——定期（或每次练习后）调用 AI 分析 `qa_history` 中的问答记录。Prompt: "分析该生本学期的错题与疑问，列出前 5 个薄弱知识点，并给出复习建议。" 前端以雷达图 (ECharts) 可视化展示。

### 4.6 英语 3500+ 智能记忆 (模块补充)
- **上传词库**：支持 CSV/TXT 上传，系统自动解析并存入 `word_list`。
- **AI 记忆法**：点击单词，调用大模型生成谐音法、词根法或联想口诀。
- **艾宾浩斯复习**：单词测试采用"看中文拼英文"或"看英文选中文"。基于 `mastery_level` 字段，利用 SM-2 算法（简单间隔重复）安排复习时间，已掌握单词计数实时更新。

## 5. 用户流程与 UI 布局要求
- **首页**：Dashboard 展示今日学习时长、待复习单词数、错题数、继续学习最近记录。
- **学习区**：三栏布局。左：章节列表；中：AI 对话流（核心）；右：小画板/手写区域（可折叠）。
- **设置页 (`/settings`)**：显眼位置放置 **API Key 输入框**（DeepSeek Key & Qwen-VL Key），并提示"Key 仅保存在本地浏览器"。

## 6. 我补充的隐藏功能 (提升自学价值)
1.  **错题本自动归档**：练习中答错的题，不仅重新讲解，还自动存入 `wrong_questions` 表。支持按学科、难度筛选，支持"重做错题"功能，系统只组错题卷。
2.  **数学公式渲染**：项目中必须集成 `KaTeX` 或 `MathJax`，确保 AI 输出的 LaTeX 公式（如 `\frac{x}{y}`）在前端正常渲染，这是理科自学的基石。
3.  **防沉迷/专注模式**：学习计时器功能，如果 5 分钟无操作，自动暂停计时，确保统计准确。

## 8. 代码规范 (防止屎山)
- 所有 API 调用必须写在 `services/` 目录下，严禁在组件内直接 fetch。
- 所有 Prompt 模板集中放在 `prompts/` 目录下，便于后期调优。
- 使用 React Suspense 处理加载状态，配合 `react-error-boundary` 捕获 AI 接口异常。
